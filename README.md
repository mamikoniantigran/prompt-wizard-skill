# Prompt Wizard Skill for Claude Code

Методология создания эффективных промптов по принципу «3 артефакта» — Reference, Blueprint, Success Brief. Превращает расплывчатые запросы к AI в точные, воспроизводимые промпты для серьёзных задач.

## Возможности

- **Методология «3 артефакта»** — структурированный подход: образец → чертёж → бриф успеха
- **Два трека** — быстрые запросы обрабатываются просто, серьёзные задачи — по методологии
- **6 правил диалога** — как эффективно общаться с AI после первого ответа
- **Диагностика промптов** — анализ почему существующий промпт не работает и как его починить
- **Мульти-LLM Deep Research** — параллельный запуск в ChatGPT, Gemini, Perplexity → кросс-чек в Claude Code → мерж без галлюцинаций
- **Универсальность** — работает с любой LLM, не только Claude

## Метод кратко

```
Prompting = Thinking, не Typing.
```

Для серьёзной задачи нужны три артефакта:

| # | Артефакт | Что это | Пример |
|---|----------|---------|--------|
| 1 | **Reference** | Конкретный образец желаемого результата | Контракт, отчёт, письмо, лендинг |
| 2 | **Blueprint** | Анализ: что делает образец эффективным (100 слов) | Тон, структура, ключевые паттерны |
| 3 | **Success Brief** | Определение успеха для ТВОЕЙ версии | Тип, реакция получателя, критерии |

Три артефакта собираются в финальный промпт → AI получает полный контекст → результат точный с первого раза.

## Quick Start

### Требования

- [Claude Code](https://docs.anthropic.com/en/docs/claude-code)

### Установка

1. Скопируйте `SKILL.md` в директорию скиллов вашего проекта:
   ```bash
   cp SKILL.md /path/to/your/project/
   ```

2. Добавьте ссылку в `CLAUDE.md` вашего проекта:
   ```markdown
   ## Skills
   - prompt-wizard: методология «3 артефакта» для создания эффективных промптов
     - Триггеры: "напиши промпт", "создай prompt", "улучши промпт", "prompt engineering"
   ```

3. Используйте:
   ```
   > напиши промпт для анализа инвестиционного контракта
   ```

## Как это работает

1. **Классификация** — скилл определяет тип запроса: быстрый или серьёзная задача
2. **Reference** — помогает найти или описать образец желаемого результата
3. **Blueprint** — анализирует образец и создаёт «чертёж» для воспроизведения
4. **Success Brief** — формулирует критерии успеха для вашей версии
5. **Сборка** — собирает три артефакта в готовый промпт с планом выполнения
