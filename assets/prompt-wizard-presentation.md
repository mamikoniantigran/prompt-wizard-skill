# Prompt Wizard
## Методология «3 артефакта» для Claude Code

**Формат**: self-paced, ~15 мин
**Аудитория**: сообщество Claude Code (от новичков до продвинутых)
**Дата**: 6 апреля 2026
**Автор**: Tigran Mamikonian

---

## Slide 1: Проблема

**Key message**: Большинство промптов не работают не потому, что AI слабый — а потому что запрос расплывчатый.

**Контент**:

Типичный промпт:
> «Напиши мне бизнес-план»

Что получает AI:
- Нет образца — непонятно, что именно нужно
- Нет контекста — для кого, зачем, какой формат
- Нет критериев — как понять, что результат хороший

Результат: generic ответ → разочарование → «AI не работает»

---

## Slide 2: Решение — Prompt Wizard

**Key message**: Prompting = Thinking, не Typing.

**Контент**:

Prompt Wizard — это методология, которая превращает расплывчатые запросы в точные промпты через три артефакта:

| # | Артефакт | Суть |
|---|----------|------|
| 1 | **Reference** | Образец желаемого результата |
| 2 | **Blueprint** | Анализ: что делает образец эффективным |
| 3 | **Success Brief** | Определение успеха для ТВОЕЙ версии |

Три артефакта = полный контекст для AI = точный результат с первого раза.

---

## Slide 3: Артефакт 1 — Reference

**Key message**: Покажи AI, что ты хочешь получить — не описывай, а покажи.

**Контент**:

**Что это**: конкретный образец результата, который тебе нравится.

**Примеры**:
- Инвестиционный отчёт, который ты видел у коллеги
- Письмо клиенту, которое сработало в прошлом
- Лендинг конкурента с хорошей конверсией
- Контракт из предыдущей сделки

**Как найти**: вспомни лучший пример того, что тебе нужно. Если нет — опиши словами, как выглядит идеальный результат.

---

## Slide 4: Артефакт 2 — Blueprint

**Key message**: Разобрать образец на составляющие — понять, ПОЧЕМУ он работает.

**Контент**:

Claude Code анализирует Reference и создаёт «чертёж» в ~100 слов:

- **Тон** — формальный, разговорный, экспертный
- **Структура** — как организован контент
- **Ключевые паттерны** — что делает образец эффективным
- **Приёмы** — конкретные техники подачи

Blueprint — это рецепт, по которому AI воспроизведёт качество образца для ТВОЕЙ задачи.

---

## Slide 5: Артефакт 3 — Success Brief

**Key message**: Определить успех ДО начала работы, а не после.

**Контент**:

| Вопрос | Пример ответа |
|--------|---------------|
| **Тип документа** | Инвестиционный отчёт для партнёров |
| **Кто получатель** | Текущие инвесторы фонда |
| **Реакция получателя** | «Понятно, прозрачно, хочу продолжать» |
| **Обязательные элементы** | ROI, риски, план на квартал |
| **Запрещено** | Маркетинговый язык, обещания доходности |

Success Brief закрывает главную проблему: AI теперь знает, что значит «хороший результат» для тебя.

---

## Slide 6: Как это работает вместе

**Key message**: Три артефакта собираются в один промпт — AI получает полный контекст.

**Контент**:

```
Reference (образец)
       |
       v
Blueprint (анализ: тон, структура, паттерны)
       |
       v
Success Brief (тип, получатель, критерии)
       |
       v
   ФИНАЛЬНЫЙ ПРОМПТ
       |
       v
  Точный результат
```

Весь процесс занимает 5-10 минут диалога с Claude Code. Результат: промпт, который работает с первого раза.

## Slide 7: Deep Research — мульти-LLM workflow

**Key message**: Для серьёзных вопросов один AI недостаточен — используй три и сверяй.

**Контент**:

Prompt Wizard включает полный workflow для Deep Research:

1. **Подготовка** — базовый промпт через 3 артефакта
2. **3 промпта** — Claude Code генерирует copy-paste ready версии для:
   - ChatGPT (Deep Research)
   - Gemini (Deep Research)
   - Perplexity
3. **Параллельный запуск** — все три одновременно, независимо
4. **Возврат результатов** — файлами или вставкой в Claude Code
5. **Кросс-чек** — сравнение, разрешение конфликтов
6. **Финальный отчёт** — только верифицированные факты

## Slide 8: Deep Research — как это выглядит

**Key message**: Каждый промпт адаптирован под сильные стороны конкретной LLM.

**Контент**:

| LLM | Специализация | Формат промпта |
|-----|---------------|----------------|
| **ChatGPT** | Глубокий анализ, структура | Академический стиль, таблицы |
| **Gemini** | Свежие данные, веб-поиск | Практический фокус, ссылки |
| **Perplexity** | Фактчекинг, источники | Прямые вопросы, верификация |

Каждый промпт содержит блок трассируемости:
- Полный текст запроса (для воспроизводимости)
- Дата и модель (для версионирования)

Результат: три независимых мнения → кросс-чек в Claude Code → один верифицированный отчёт без галлюцинаций.

## Slide 9: Deep Research — сохранение и архив

**Key message**: Исследование без архива = потерянное исследование. Сохраняй всё.

**Контент**:

**Что сохраняется:**

| Файл | Содержимое | Зачем |
|------|------------|-------|
| `chatgpt_research.md` | Полный ответ ChatGPT | Исходник для воспроизводимости |
| `gemini_research.md` | Полный ответ Gemini | Исходник для воспроизводимости |
| `perplexity_research.md` | Полный ответ Perplexity | Исходник для воспроизводимости |
| `{тема}_deep_research_{дата}.md` | Финальный мерж | Верифицированный результат |

**Структура папки проекта:**
```
project/
├── research/
│   ├── chatgpt_research.md      ← исходники (не удалять!)
│   ├── gemini_research.md
│   ├── perplexity_research.md
│   └── investment_analysis_deep_research_2026-04-06.md  ← финальный отчёт
```

**Источники**: все URL из трёх отчётов объединяются в пронумерованный список в финальном документе. Дубликаты убираются, битые ссылки помечаются.

Исходники хранятся бессрочно — для повторного анализа, аудита или обновления через месяцы.

## Slide 10: Два трека — быстрый и серьёзный (выбор трека)

**Key message**: Не каждый запрос требует методологии — скилл это понимает.

**Контент**:

| | Быстрый трек | Серьёзный трек |
|---|---|---|
| **Когда** | Простые задачи | Высокие ставки |
| **Примеры** | Письмо, список, перевод | Контракт, отчёт, стратегия |
| **Метод** | Сразу ответ | 3 артефакта |
| **Deep Research** | Нет | Опционально |
| **Время** | 1 мин | 5-15 мин |

Claude Code автоматически классифицирует запрос и предлагает подходящий трек.

## Slide 11: Quick Start

**Key message**: Установка за 2 минуты — один файл, одна строка в CLAUDE.md.

**Контент**:

**Шаг 1**: Скопировать SKILL.md в проект
```bash
cp SKILL.md /path/to/your/project/
```

**Шаг 2**: Добавить в CLAUDE.md

## Skills

**Шаг 3**: Использовать
