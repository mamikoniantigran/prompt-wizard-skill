# Prompt Wizard

Методология создания эффективных промптов для серьёзных задач.

## Ключевой принцип

**Prompting = Thinking, не Typing.**

LLM не знает что у тебя в голове — она знает только что ты написал. Расплывчатый ввод = расплывчатый результат.

## Два типа запросов

1. **Быстрые запросы** — как поиск в Google. Низкие ставки, простая задача. Сложный промпт НЕ нужен.

2. **Серьёзные задачи** — брейншторм, анализ контракта, маркетинговый план, бриф. Требуют методичного подхода.

Этот скилл для второго типа.

## Методология: 3 артефакта

Для серьёзной задачи нужны три вещи:

### 1. Reference (Образец)

Конкретный пример желаемого результата — контракт, отчёт, письмо, лендинг.

**Процесс:**
- Найти образец, который вызывает реакцию "хочу именно так"
- Конвертировать в markdown (если нужно)
- Загрузить в чат или проект

### 2. Blueprint (Чертёж)

Анализ того, ЧТО делает образец эффективным.

**Промпт для создания blueprint:**
```
Проанализируй этот reference чтобы потом воссоздать подобное.
Дай краткий, действенный чертёж:
- Что это?
- Тон
- Ключевые паттерны

Уложись в 100 слов. Это твои инструкции для воссоздания подобного текста без доступа к оригиналу. Всё в codeblock.
```

### 3. Success Brief (Бриф успеха)

Определение успеха для ТВОЕЙ версии:

```
SUCCESS BRIEF

Тип и объём: [Контракт, отчёт, письмо? Сколько страниц?]
Реакция получателя: [Что должен подумать/почувствовать/сделать после прочтения?]
НЕ должно звучать как: [Чего избегать — generic AI, слишком формально, жаргон?]
Успех это: [Подпишут? Одобрят? Ответят? Примут решение?]
```

## Сборка финального промпта

```
[1. Загрузить reference файл]

Я загрузил образец того, что хочу получить. Вот что делает его эффективным:

###
[2. Вставить blueprint]
###

Вот что нужно для моей версии:

###
[3. Вставить success brief]
###

Теперь создай план выполнения по шагам (максимум 5). Определи структуру и задай один вопрос, чтобы начать первый шаг.
```

## Правила диалога с AI

После первого ответа — ОБЩАЙСЯ с AI. Это критически важно.

**6 правил эффективного диалога:**

1. **Исправляй ошибки сразу** — "Это неверно — X на самом деле Y потому что [причина]. Исправь и продолжай."

2. **Каждое сообщение кратко** — Без предисловий и любезностей. Только: исправление (если нужно) + следующая инструкция.

3. **Ссылайся на предыдущее точно** — "В предыдущем ответе ты сказал [цитата]. Разверни это добавив..."

4. **Используй директивный язык** — "Ответь кратко", "Один абзац", "Буллеты без воды", "Переделай до 200 слов".

5. **Агрессивно двигайся к финалу** — "Теперь создай финальную версию со всеми исправлениями" или "Собери всё в чистый итоговый ответ".

6. **Перезапускай при раздутии контекста** — После 8-10 обменов или при падении качества: скопируй лучшую версию → новый чат → "Вот текущая версия — улучши/доверши..."

## Workflow

При получении запроса на создание промпта:

1. **Уточни задачу**: Это быстрый запрос или серьёзная задача?

2. **Для быстрых запросов**: Напиши простой, прямой промпт без усложнений.

3. **Для серьёзных задач**:
   - Спроси есть ли у пользователя reference (образец)
   - Помоги создать blueprint если есть образец
   - Помоги сформулировать success brief
   - Собери финальный промпт

4. **Для улучшения промпта**:
   - Проанализируй текущий промпт
   - Определи что конкретно не работает (расплывчато? нет критериев успеха? всё в один заход?)
   - Примени методологию 3 артефактов

5. **Предложи правила диалога** если задача многошаговая.

## Deep Research: мульти-LLM workflow

Для глубоких исследовательских вопросов — использовать параллельный запуск в нескольких LLM с последующим кросс-чеком. Одна нейронка может галлюцинировать — три нейронки дают верифицированный результат.

### Шаг 1. Подготовка промпта

Применить методологию 3 артефактов к исследовательскому вопросу:
- **Reference**: пример хорошего исследования по похожей теме (формат, глубина, структура)
- **Blueprint**: что делает это исследование качественным
- **Success Brief**: что конкретно нужно узнать, в каком формате, для какого решения

### Шаг 2. Адаптация под каждую LLM

Один базовый промпт → три версии с учётом сильных сторон:

| LLM | Сильные стороны | На что делать акцент |
|-----|-----------------|---------------------|
| **ChatGPT Deep Research** | Глубокий анализ, длинные отчёты, browses web | Комплексный обзор, множественные источники |
| **Gemini Deep Research** | Свежие данные Google, структурированные отчёты | Актуальные данные, статистика, тренды |
| **Perplexity** | Быстрый поиск с источниками, цитирование | Факты с прямыми ссылками на источники |

### Шаг 3. Блок трассируемости (обязательно)

В конец КАЖДОГО промпта добавлять:

```
В конце ответа обязательно укажи:
1. **Промпт**: процитируй полный текст этого промпта
2. **Дата исследования**: [текущая дата]
3. **Модель**: [название LLM и режим, например "ChatGPT Deep Research"]
4. **Источники**: пронумерованный список всех использованных источников с URL
```

**Зачем**: результаты deep research сохраняются, пересылаются, цитируются через недели/месяцы. Без метаданных невозможно оценить актуальность и воспроизвести исследование.

### Шаг 4. Параллельный запуск

Отправить промпт одновременно в 2-3 LLM. Не ждать ответа одной, чтобы корректировать промпт для другой — важно получить **независимые** результаты.

### Шаг 5. Кросс-чек и мерж в Claude Code

После получения всех ответов — загрузить их в Claude Code (или Claude с длинным контекстом) с промптом:

```
У меня три независимых исследования одного вопроса от разных LLM.
Задача:

1. **Кросс-чек фактов**: найди утверждения, которые подтверждаются
   минимум двумя источниками. Пометь как ✅ Verified.

2. **Выяви расхождения**: найди противоречия между отчётами.
   Для каждого противоречия укажи:
   - Что говорит каждая LLM
   - Какая версия вероятнее и почему
   - Пометь как ⚠️ Conflict

3. **Флаг галлюцинаций**: найди утверждения, которые есть только
   в одном отчёте и не подтверждены источниками.
   Пометь как ❌ Unverified.

4. **Мерж**: создай единый отчёт, включающий только
   верифицированные факты + разрешённые конфликты.
   В конце — секция «Требует дополнительной проверки»
   для unverified утверждений.

Отчёты:

### ChatGPT Deep Research
[вставить]

### Gemini Deep Research
[вставить]

### Perplexity
[вставить]
```

### Когда использовать мульти-LLM workflow

- Решения с высокими ставками (инвестиции, юридические вопросы, стратегия)
- Темы, где данные быстро устаревают (рынки, регулирование, технологии)
- Вопросы, где одна LLM может не иметь полной информации
- Когда нужна максимальная достоверность для принятия решения

### Когда НЕ нужно

- Быстрые фактчеки (достаточно одной LLM)
- Творческие задачи (субъективные, нет «правильного» ответа)
- Вопросы с однозначным ответом

## Частые ошибки (что НЕ делать)

- Расплывчатые описания без конкретики
- Отсутствие определения успеха
- Попытка уместить всё в один промпт
- Надежда что AI "поймёт" что ты имеешь в виду
- Игнорирование диалога после первого ответа
